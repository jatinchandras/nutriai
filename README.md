# NutriAI — AI Calorie Tracker

**Stack:** React + Vite (Vercel) · Node.js/Express (Vercel Serverless) · Supabase (Postgres + Google Auth) · Groq + Llama 3 (free AI)

**PWA:** Installable on Android/iOS. Play Store publishable via PWABuilder.

---

## 🔑 API Keys Needed (all free, no credit card)

| Service | Where to get it | Used for |
|---------|----------------|----------|
| Supabase | supabase.com → New Project | Database + Google Auth |
| Groq | console.groq.com → API Keys | AI food/workout estimation |
| Google OAuth | console.cloud.google.com | Sign-in |

## 🚀 Deploy in ~15 Minutes

### Step 1 — Supabase setup (5 min)

1. Create a free project at [supabase.com](https://supabase.com)
2. Go to **SQL Editor** → paste and run `supabase/migrations/001_init.sql`
3. Go to **Authentication → Providers → Google** → enable it
   - Create OAuth credentials at [console.cloud.google.com](https://console.cloud.google.com)
   - APIs & Services → Credentials → Create → OAuth 2.0 Client ID → Web
   - Authorized redirect URI: `https://YOUR_PROJECT.supabase.co/auth/v1/callback`
4. Note your **Project URL**, **anon key**, **service_role key** (Settings → API)

### Step 2 — Get Groq API key (2 min)

1. Go to [console.groq.com](https://console.groq.com) → sign in with Google
2. API Keys → Create new key → copy it
3. Free tier: 14,400 requests/day, 30 requests/min — more than enough

### Step 3 — Generate PWA icons (2 min)

```bash
cd frontend
npm install
node scripts/generate-icons.js
```

This creates all required icon sizes in `public/icons/`.
Replace them with your own branding if desired.

### Step 4 — Push to GitHub (1 min)

```bash
git init && git add . && git commit -m "initial"
git remote add origin https://github.com/YOUR_USERNAME/nutriai.git
git push -u origin main
```

### Step 5 — Deploy Frontend on Vercel (4 min)

1. [vercel.com](https://vercel.com) → New Project → import your repo
2. **Root Directory:** `frontend`
3. **Framework:** Vite (auto-detected)
4. **Environment Variables:**
```
VITE_SUPABASE_URL=https://YOUR_PROJECT.supabase.co
VITE_SUPABASE_ANON_KEY=your_anon_key
VITE_API_URL=https://your-backend.vercel.app/api
```
5. Deploy → note your frontend URL (e.g. `https://nutriai.vercel.app`)

### Step 6 — Deploy Backend on Vercel (3 min)

1. New Project → same repo, **Root Directory:** `backend`
2. **Environment Variables:**
```
SUPABASE_URL=https://YOUR_PROJECT.supabase.co
SUPABASE_ANON_KEY=your_anon_key
SUPABASE_SERVICE_ROLE_KEY=your_service_role_key
GROQ_API_KEY=gsk_your_groq_key
FRONTEND_URL=https://nutriai.vercel.app
NODE_ENV=production
```
3. Deploy → note your backend URL
4. Go back to **frontend** project → update `VITE_API_URL` to your backend URL

### Step 7 — Supabase Auth redirect (1 min)

Supabase → Authentication → URL Configuration:
- **Site URL:** `https://nutriai.vercel.app`
- **Redirect URLs:** `https://nutriai.vercel.app/auth/callback`

## 🏃 Run Locally

```bash
# Frontend
cd frontend
cp .env.example .env.local   # fill in values
npm install
node scripts/generate-icons.js   # one-time icon generation
npm run dev    # → http://localhost:5173

# Backend (separate terminal)
cd backend
cp .env.example .env   # fill in values
npm install
npm run dev    # → http://localhost:3001
```

## 💡 How Sessions Work

- Users sign in with Google → Supabase creates a session JWT
- Session lasts **7 days** and auto-refreshes silently in the background
- Users almost never need to re-login
- To extend to 30 days: Supabase → Auth → Settings → JWT expiry → 2592000

---

## 🔒 Security Notes

- API keys are **never** in the frontend — only `VITE_SUPABASE_URL` and `VITE_SUPABASE_ANON_KEY` (both safe to expose)
- The Groq key and `SUPABASE_SERVICE_ROLE_KEY` live only on the backend (Vercel env vars)
- Every API route verifies the user's Supabase JWT before doing anything
- Supabase Row Level Security means users can only ever read/write their own rows
